### **# Core Concepts & Terminology (from docs)**

*   **PayPal JavaScript SDK**: Client-side library for embedding PayPal buttons and handling payments securely in the browser. ([javascript-scrape.json] `/sdk/js/`)
    *   Key Component: `CardFields` (for accepting card details securely). ([javascript-scrape.json] `/sdk/js/v1/reference/`)
    *   Key Component: `HostedFields` (Legacy v5 component for secure card entry). ([javascript-scrape.json] `/sdk/js/v1/reference/`)
    *   Relevant Files: `paypal-js` npm package, `react-paypal-js` npm package. ([javascript-scrape.json] `/sdk/js/`)
*   **PayPal REST APIs**: Server-side APIs for payment creation, authorization, capture, refunds, subscriptions, etc. ([pp-rest-api.json] `/api/rest/`)
    *   **Orders API**: Creates payment orders, handles intent (capture/auth). ([jacascript-scrape.json], [pp-rest-api.json])
    *   **Payments API**: Retrieves details about sales, authorizations, captures, refunds. ([jacascript-scrape.json] line 338)
    *   **Subscriptions API**: Manages recurring payments. ([jacascript-scrape.json] `/docs/api/subscriptions/v1/`)
*   **Webhooks**: PayPal sends server-to-server notifications about events (e.g., payment completed, refund processed). ([jacascript-scrape.json] line 100, `/docs/payouts/standard/reference/webhooks/`)
*   **Authentication**: OAuth 2.0 client credentials for server-side REST API calls. ([pp-rest-api.json] `/api/rest/`)
*   **Sandbox vs Live**: Separate environments for testing and production. ([pp-rest-api.json] `/api/rest/`)
*   **Security Guidelines**: Includes CSRF, XSS prevention, vulnerability management, secure development process. ([jacascript-scrape.json] `/api/rest/reference/info-security-guidelines/`, lines 14-20, 118-125)
